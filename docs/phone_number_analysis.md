# 电话号码方案评估（选型决策记录）

> 题目 FAQ 原文："电话号码怎么解决？这本身就是考察的一部分……遇到困难请记录下来，答辩时讨论"，并允许"用你自己的电话作为 Demo"。本文记录评估过程与决策依据。

## 1. 需求约束

- 访客拨打一个号码 → AI Voice Agent 实时接听并对话（双向音频）。
- Demo 阶段可用自己的电话拨打。
- 一个人，约 7 天，最小可本地部署。

## 2. 核心结论

个人在 7 天内、走合规渠道，**拿不到一个能被 AI 实时接听的中国大陆 +86 号码**。
根本原因：大陆号码资源由运营商强监管发放，把 +86 接到"程序/AI 接听"属于电信增值业务，必须企业资质 + 运营商报备审核。这道墙与使用哪个 AI 无关——即便用国产 AI 做对话层，绑定 +86 呼入号仍卡在同一资质上。

## 3. 候选方案与判断

| 方案 | 能否拿到可用号码 | 个人/7天可行性 | 判断 |
|---|---|---|---|
| 阿里云语音服务 + 智能语音机器人 | +86，需企业资质 | 否 | 明确不支持个体户/个人；多日审核+月租 |
| Twilio 导入号码 | 无 +86（大陆不在目录） | 否 | 拿不到中国号 |
| 400 / 95 号码 | +86，呼入 | 否 | 400 需营业执照；95 需呼叫中心许可证(注册资本千万级) |
| 个人手机卡 + GOIP 网关桥接 | +86 | 违法 | 《反电信网络诈骗法》明令禁止，属"电诈工具"，可能构成帮信罪 |
| 香港 +852（SIP/DID 提供商）+ BYO SIP 接 Vapi | +852 | 可能（也可能需 KYC） | "中华区"号码、可在大陆拨打，作为可选升级 |
| **Vapi 海外号（美国免费）+ 自己手机拨打** | +1 | **是** | **demo 主线，零阻塞** |

## 4. 证据

- 阿里云语音服务：必须企业实名 + 单独"企业资质审核"，白纸黑字"个体工商户、自然人独资、个人独资公司不支持使用语音服务"；号码预审 1 工作日 + 出号 2~10 工作日 + 话术报备。
  - https://help.aliyun.com/zh/vms/user-guide/enterprise-qualification-management
  - https://help.aliyun.com/zh/vms/user-guide/real-number-application
- Twilio 不提供 +86 号码（大陆不在可开通目录）。
- 400/95 号码个人不可办；400 需营业执照且仅呼入；95 需《全国呼叫中心许可证》。
  - http://www.dttc-gw.com/art/23192.html
  - https://www.400link.cn/share/knowledge/individual-businesses-will-handle-400-hotline-in-2026.html
- GOIP 桥接违法：《反电信网络诈骗法》第十四条；工信部已不发新入网许可；常被认定为电诈工具。
  - https://zh.wikipedia.org/zh-cn/GoIP
  - https://www.xinhuanet.com/legal/2022-02/15/c_1128376599.htm

## 5. 最终决策

- **Demo 主线**：网页通话（Vapi Web Call）跑通完整闭环：网页/电话 → Vapi 中文 Agent → 后端 → 微信。
  - 注：Vapi 免费美国号仅限美国境内使用，接不了中国/新加坡的国际来电（已验证），故 demo 不依赖它。
- **真实电话（可选增强）**：
  1. 甲方提供测试呼入号（见第 7 节），把回调指向本服务；或
  2. 香港 +852 DID（如 DIDHub，OFCA 无需 KYC、约 $4/月）经 BYO SIP 接入 Vapi，中国大陆手机可直拨。
- **生产演进路线**（写入 README）：取得企业资质（营业执照+公章）后走阿里云 +86 呼入号，或经持牌 SIP/DID 提供商 BYO SIP 接入。
- **合规红线**：不使用 GOIP / 改号等违规设备。

## 6. 答辩要点（一句话版）

"号码资源是强监管的电信增值业务，个人 7 天内拿不到合规 +86 AI 呼入号；我评估了阿里云、Twilio、400/95、GOIP、香港号五条路并给出依据，demo 用网页通话跑通闭环，真实电话用甲方测试号或香港 SIP 升级——这套判断本身就是题目要考的技术筛选与方案评估。"

## 7. 向甲方问询测试呼入号（已发起）

个人无法获得合规 +86 呼入号，但甲方是 AI 公司、很可能已有云语音资质或现成号码。已主动沟通，请其在方便时提供一个测试呼入号（把回调指向本服务）；否则用网页通话演示闭环。这是把"资质阻塞"转成"找资源方"的现实推进，也是答辩可讲的沟通点。

沟通话术（实际发出版）：

> 您好，我今天看了下题目在做可行性调研。Voice Agent 的通路验证了一下——中文语音对话、信息采集、微信通知基本能串起来。
> 唯一卡住的是电话号码：合规的 +86 号要企业电信资质和运营商报备，我个人短期内确实搞不到一个能让 AI 接听的号。
> 想问下你们手头有没有现成、可对外提供的测试呼入号，能给我接一下、把回调指到我的服务？有的话真打电话的 demo 才有跑通的可能性。
