# 015 车牌省份首字：拼音兜底纠错（仅唯一命中才纠）

## 背景 / 约束
- 6/21 朋友反馈：车牌**省份单字**(沪/苏/皖…)STT 识别率偏低，常被听成同音字(湖、户、速…)。
- 既然公司名能做拼音模糊匹配(决策 014)，省份简称是**封闭 31 值**，更适合算法化纠错。
- 原有 `_PROVINCE_CONFUSABLE` 只是「手工堆近音字」——和决策 014 否决的方案 A 同病：脆、覆盖不全、遇新变体就得加。
- 关键约束：**车牌是开集，猜错＝静默写错牌、没人发现**(不像公司有白名单兜底)。单字拼音有不可消除歧义：桂/贵完全同音；京 jīng / 津 jīn / 晋 jìn 相近。所以纠错可以激进扩召回，但**绝不能在歧义时乱猜**。

## 候选方案
- A. **维持手工近音表**：现状，靠人列。漏召回，不可持续。
- B. **改 prompt 让司机报「上海的沪」给 STT 上下文**：用户明确否决——「这种东西不可能写在 prompt 里还让司机说」，且治标不治本(司机不会照说)。
- C. **算法化拼音兜底（本次采用）**：把听到的非省份首字转拼音，与 31 省读音比对，**只在唯一命中一个省时才纠**，命中 0 个或多个一律不纠、保留原样交给校验/对话重说。

## 最终选择
选 **C**，叠加在手工表之上(手工表含形近字/多音字如「了→辽」拼音兜不住，保留作第一层)。`clean_plate` 纠错链：**手工近音表 → 拼音兜底(唯一命中) → 原样**。

拼音兜底两级、均「唯一命中才纠」：
1. **带声调**唯一索引：分得清 京 jing1 / 津 jin1 / 晋 jin4(襟→津、禁→晋、葬→藏)。
2. 不中再走**无声调**唯一索引：容忍 STT 听错声调(湖 hú→沪 hù、庐 lú→鲁、扇 shàn→陕)。

歧义一律返回 None 不猜：带声调 `gui4`(桂/贵)；无声调 `gan/gui/ji/jin/yu`(赣甘/桂贵/冀吉/津晋/豫渝)。

## 实现
- `plate_registry.py`：
  - 加 `from pypinyin import Style, pinyin`(复用决策 014 已引入的 `pypinyin==0.55.0`)。
  - 硬编码 `_PROVINCE_PINYIN`(31 省 TONE3 读音)——单字多音，省份侧不靠 pypinyin 猜，与 heard 字同制式(TONE3)比对。
  - `_unique_pinyin_index()` 建索引时丢弃多省共享键，从根上杜绝错猜；得 `_TONE_TO_PROVINCE`(29) 与 `_TONELESS_TO_PROVINCE`(21)。
  - `_province_by_pinyin()`：带声调命中→无声调命中→None。
  - `clean_plate()` 纠错行接入：`_PROVINCE_CONFUSABLE.get() or _province_by_pinyin() or 原值`。
- 测试：`test_province_pinyin_fallback`(庐→鲁、襟→津、禁→晋、扇→陕、葬→藏，均不在手工表，走拼音)；`test_province_pinyin_ambiguous_not_guessed`(跪/鸡/宇 → `_province_by_pinyin` 为 None 且 `normalize_plate` 抛 `InvalidPlateError`)。本地 `pytest` 37 绿。

## 理由
- 用算法替代手工别名，覆盖 STT 同音错字的主要模式，符合「省份是封闭集」的结构。
- 「唯一命中才纠」把开集车牌的静默错纠风险降到 0：宁可不纠、走重说，也不冒充对的。

## 已知代价 / 可演进
- 真同音歧义(桂/贵、津/晋 听错声调时)纠不了 → 留给对话「我听到的是 X，对吗」重说，是安全取舍。
- pypinyin 取 heard 字默认读音，个别多音字(如「了」)拼音兜不住 → 由手工表第一层补，二者互补。
- 可演进：需要时把 heard 字也按 `heteronym=True` 全读音匹配(仍要求跨读音唯一)以再扩召回。
